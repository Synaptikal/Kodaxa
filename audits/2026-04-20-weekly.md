## Exhaustive-Deps Review — Awaiting Approval

I reviewed the three `react-hooks/exhaustive-deps` warnings and propose the following non-destructive fixes. I did NOT apply these changes; they are ready for your approval.

1) File: `src/components/building/building-shell.tsx` — effect: "On mount, load state from URL if present"
- Current deps: `[searchParams]`
- Problem: effect calls `hook.loadState(...)` but `hook.loadState` was not included in deps; this can cause a stale-closure if the hook instance changes.
- Proposed fix: change deps to `[searchParams, hook.loadState]`.
- Rationale: `loadState` is a stable callback returned by `useBuildingState` (declared with `useCallback`), so adding it is safe and ensures the latest implementation is used without re-running unnecessarily.

2) File: `src/components/building/building-shell.tsx` — effect: "Sync state back to URL when layout changes"
- Current deps: `[hook.state.cells, hook.state.name, hook.state.activeLayer, hook.state.claimX, hook.state.claimZ]`
- Problem: the effect also references `generateBuildingShareUrl(hook.state, ...)` (uses full `hook.state`) and checks `searchParams.size`, but `hook.state` (the whole object) and `searchParams.size` were not listed.
- Proposed fix: change deps to include the state object and the params size: `[hook.state, searchParams.size]` (or equivalently include `hook.state` plus the existing specific state fields and `searchParams.size`).
- Rationale: `generateBuildingShareUrl` consumes the whole `hook.state`; listing the whole `hook.state` ensures the effect runs whenever relevant state changes. Including `searchParams.size` makes the mount-guard deterministic when URL params change.

3) File: `src/components/planner/planner-shell.tsx` — hook: `useMemo` decoding `initialBuild`
- Current deps: `[]` with a comment: "Only decode once on mount — intentional empty deps"
- Problem: ESLint correctly points out `searchParams` is used but not listed. Here the empty deps are intentional to avoid re-decoding and overwriting user edits when URL params change after mount.
- Proposed fix: keep the empty deps but add a clarifying inline disable and reason, e.g.:
	// eslint-disable-next-line react-hooks/exhaustive-deps -- decode only once on mount to avoid overwriting user edits when URL updates
	const initialBuild = useMemo(() => { ... }, []);
- Rationale: decoding only at mount is an explicit design choice; adding the single-line disable documents the intention and satisfies lint rules while preserving behavior.
